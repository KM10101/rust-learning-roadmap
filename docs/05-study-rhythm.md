# 学习节奏

## 推荐频率

可持续的学习节奏，比偶尔爆发式投入更重要。

建议基线：

- 每周 3 到 5 次学习
- 每次 30 到 60 分钟
- 至少 2 次必须是亲手写代码

## 推荐的每周结构

### Session A
- 阅读或复习一个概念
- 写几个极小的例子

### Session B
- 继续推进一个小项目
- 聚焦一个反复卡住的痛点

### Session C
- 回看之前遇到的编译器报错
- 用更干净的方式重写一个解法

### Session D 或 E
- 阅读真实项目中的地道代码
- 和自己的代码做对比

## 要避免的陷阱

- 读得太多，写得太少
- 写得太多，却缺少复盘
- 抄了解法，却没理解所有权决策
- 过早跳进高级 async 代码

## 比“学会 Rust”更好的目标

建议把目标写成这样：

“能够有信心地用 Rust 做出一个小而有用的工具，以及一个小型后端服务。”

这个目标更具体，也更容易衡量。
