# 学习路线图

## 阶段 1：重新上手与基础恢复

目标：重新建立对 Rust 代码的阅读、运行和轻微修改能力。

重点包括：

- cargo 基础
- 变量、函数、结构体、枚举
- 模式匹配
- `Option` 与 `Result`
- 所有权与借用基础
- 模块与 crate

阶段结果应当是：

- 能运行小型 Rust 程序
- 能在基础层面理解编译器反馈
- 能完成简单练习而不慌乱

## 阶段 2：真正理解核心模型

目标：不再把所有权看成“玄学”。

重点包括：

- move 与 borrow 的区别
- 可变引用与不可变引用
- slice 与字符串
- 集合类型
- 使用 `?` 进行错误传播
- trait 与泛型
- 在实用层面理解生命周期

阶段结果应当是：

- 能解释借用检查错误为什么出现
- 能更有意识地在拥有值和引用之间做选择
- 能写出中等规模的练习代码，并减少盲目试错

## 阶段 3：做真实的小项目

目标：通过完整程序建立信心。

重点包括：

- CLI 工具
- 文件 IO
- 解析逻辑
- HTTP 客户端
- 使用 serde 做序列化
- 项目组织方式
- 测试

阶段结果应当是：

- 完成 3 到 5 个小型 Rust 项目
- 能交付可用的命令行工具
- 理解常见 crate 的使用模式

## 阶段 4：面向后端的 Rust

目标：把 Rust 和已有的后端工程直觉连接起来。

重点包括：

- async / await
- Tokio 运行时
- Axum 或 Actix Web
- 配置与环境管理
- 错误设计
- 日志与可观测性
- 数据库集成

阶段结果应当是：

- 能做一个小型后端服务
- 能暴露 HTTP API
- 能把项目分层组织清楚
- 能理解异步执行和共享状态

## 阶段 5：向生产级工程能力提升

目标：从“学习项目”迈向“可靠工程实践”。

重点包括：

- 性能分析
- 基于 trait 的架构抽象
- 并发模式
- 测试策略
- 向更地道 Rust 代码的重构
- 阅读成熟 Rust 开源项目

阶段结果应当是：

- 有能力把 Rust 用在严肃的工具或服务工作中
- 知道如何持续提升，而不是停留在入门阶段
