# Rust 与后端工程师

## 你已有的可迁移优势

如果你已经在使用 Python 和 Java 工作，其实你已经具备很多可迁移能力：

- API 设计思维
- 分层与模块化能力
- 错误处理纪律
- 数据建模直觉
- 测试意识
- 后端权衡判断力

## Rust 带来的变化

### 和 Python 相比

Rust 会要求你更加显式：

- 类型更居中
- 所有权是核心问题
- 可变性控制更严格
- 很多运行期错误会被前移到编译期约束

### 和 Java 相比

Rust 往往显得更轻，但同时也没有那么多重型框架保护。
你会更主动地做组合，更依赖类型系统和 crate，而不是框架魔法。

## 适合早期上手的后端栈

对面向后端的 Rust 学习来说，一个很实用的早期组合是：

- `tokio`：异步运行时
- `axum`：HTTP 服务
- `serde`：序列化
- `reqwest`：HTTP 客户端
- `tracing`：日志与可观测性
- `sqlx` 或 `sea-orm`：等基础更稳后再引入

## 建议的里程碑顺序

1. 先建立 CLI 习惯
2. 学会地道的错误处理
3. 理解异步基础
4. 做一个小型 Web API
5. 加入持久化和测试
6. 继续往更干净的抽象重构

## 长期方向

真正有用的长期目标，不只是“会 Rust 语法”。

而是：当问题确实适合 Rust 时，你能有意识地选择它。

例如当你需要这些特性时：

- 性能
- 安全性
- 并发纪律
- 系统代码的长期可维护性
