# Rust Learning Roadmap

A practical, engineering-focused roadmap for learning Rust from a backend engineer's perspective.

## Goal

This repository is designed to help the learner move from:

- having seen Rust before
- forgetting most of the details
- to being able to build real backend and tooling projects confidently

## Repository Structure

- `docs/01-why-rust.md` — why Rust is worth learning
- `docs/02-roadmap.md` — staged roadmap from basics to real projects
- `docs/03-core-concepts.md` — ownership, borrowing, lifetimes, traits, enums, errors
- `docs/04-project-ladder.md` — project sequence for progressive learning
- `docs/05-study-rhythm.md` — a sustainable study cadence
- `docs/06-rust-for-backend-engineers.md` — mapping Rust to existing backend experience

## Learning Strategy

This repo assumes the best way to learn Rust is not pure theory first, but:

1. learn enough fundamentals to read and run code
2. build small complete projects
3. revisit core concepts in context
4. gradually move toward backend services and tooling

## Success Standard

You should eventually be able to:

- read idiomatic Rust without panic
- write small CLI and service programs
- understand compiler feedback instead of fighting it blindly
- build a small backend service in Rust
- use Rust intentionally rather than just experimentally
